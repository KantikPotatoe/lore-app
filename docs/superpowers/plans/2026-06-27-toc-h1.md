# TOC Includes H1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Include `h1` headings in the page Table of Contents, indenting by depth relative to the shallowest level present so existing H2/H3-only pages look unchanged.

**Architecture:** Extract the TOC's slug-with-dedup and a new relative-depth helper into a pure, unit-tested `src/toc.ts` (mirroring `src/html.ts`), then change `TableOfContents.tsx` to scan `h1, h2, h3`, use those helpers, and render `toc-depth-{0|1|2}` classes; swap the corresponding CSS rule.

**Tech Stack:** TypeScript, React, Vitest. No new dependencies.

## Global Constraints

- `slugifyHeadings` must reproduce the current id logic byte-for-byte (lowercase → non-alphanumerics to `-` → trim leading/trailing `-` → empty becomes `heading`; the Nth repeat of a base slug, N≥1, gets a `-${count}` suffix) so existing in-page anchors keep resolving.
- Indentation is relative: `depth = level − min(levels present)`; depth 0 = base 8px (unchanged), depth 1 = 20px, depth 2 = 32px.
- Scan exactly `h1, h2, h3` (not H4–H6). The "render only when >3 headings" threshold and the IntersectionObserver/scroll behavior are unchanged.
- Run `npm run lint`, `npm run build`, and `npm run test:run` before claiming done. Single test file: `npm run test:run -- src/toc.test.ts`.
- Commit after each task; do not push.

```ts
// Pure helpers for the page Table of Contents. No DOM or React here so the
// heading-id and indentation logic can be unit-tested in isolation.

/** Slugify heading texts into unique element ids, de-duplicating repeats with a
 *  numeric suffix (the 2nd "Notes" becomes "notes-1"). One id per input, order
 *  preserved; empty / punctuation-only text slugs to "heading". */
export function slugifyHeadings(texts: string[]): string[] {
  const seen = new Map<string, number>()
  return texts.map((t) => {
    const base =
      t.trim().toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '') || 'heading'
    const count = seen.get(base) ?? 0
    seen.set(base, count + 1)
    return count === 0 ? base : `${base}-${count}`
  })
}

/** Visual nesting depth of each heading relative to the shallowest level
 *  present: depth = level − min(levels). Empty input → empty output. */
export function relativeDepths(levels: number[]): number[] {
  if (levels.length === 0) return []
  const min = Math.min(...levels)
  return levels.map((l) => l - min)
}
```

## Notes for the implementer

- **Byte-for-byte slug match matters:** `slugifyHeadings` uses the exact regex chain the component used, so previously-generated anchor ids are unchanged — old `#some-heading` links keep working.
- **`level` is intentionally dropped** from `TocEntry`: it was only used to pick the indent class, now derived from `depth`. The transient `levels` array feeds `relativeDepths` and is not stored.
- **No component test:** `TableOfContents` is measurement/observer-driven and has never had a unit test; its wiring is covered by lint + build. The extracted helpers carry the logic that's worth testing.
